Interaktywne Przykłady

Każdy przykład możesz uruchomić i modyfikować. Sprawdź jak naprawdę działają pętle, funkcje i moduły.

for Pętla for — range()
Pętla for wykonuje blok kodu określoną liczbę razy. range(n) generuje liczby od 0 do n-1.
for i in range(5):
print(i)
↑for i in — zmienna i przyjmuje kolejne wartości
↑range(5) → generuje: 0, 1, 2, 3, 4
💡 range(start, stop, step) — np. range(1, 10, 2) → 1, 3, 5, 7, 9
for Iteracja po liście
for może iterować bezpośrednio po elementach listy, napisu lub dowolnej kolekcji.
owoce = ["jabłko", "gruszka", "śliwka"]
for owoc in owoce:
print(owoc)
↑owoc — kolejno: "jabłko", "gruszka", "śliwka"
💡 Możesz iterować też po napisie: for litera in "Python"
while Pętla while
while wykonuje blok dopóki warunek jest prawdziwy. Uważaj na pętle nieskończone!
n = 1
while n <= 5:
print(n)
n += 1
↑n += 1 — bez tego pętla działa w nieskończoność!
💡 while True: + break = pętla nieskończona z wyjściem
break/continue break i continue
break — natychmiast kończy pętlę. continue — pomija resztę iteracji i przechodzi do następnej.
for i in range(10):
if i == 3: continue # pomiń 3
if i == 6: break # stop na 6
print(i)
💡 Na maturze często pytają o wynik działania pętli z break/continue — przeanalizuj każdy krok!
enumerate enumerate() — indeks + wartość
enumerate() zwraca pary (indeks, wartość) — nie musisz ręcznie liczyć indeksu.
💡 enumerate(lista, start=1) — zacznij numerację od 1
comprehension List Comprehension
Zwięzły sposób tworzenia list. Zastępuje pętlę for + append().
# bez comprehension:
kwadraty = []
for x in range(6): kwadraty.append(x**2)
# z comprehension:
kwadraty = [x**2 for x in range(6)]
💡 Schemat: [wyrażenie for x in iterable if warunek]
if/elif/else Warunki — if / elif / else
Python wykonuje pierwszy blok, którego warunek jest True. Reszta zostaje pominięta.
ocena = 75
if ocena >= 90:
print("Celujący")
elif ocena >= 70:
print("Dobry")
else:
print("Niedostateczny")
💡 Zmień wartość ocena na np. 95 lub 50 i sprawdź wynik!
and/or/not Operatory logiczne
and — oba muszą być True. or — przynajmniej jeden True. not — negacja.
💡 Skrót: 10 < x < 20 działa tak samo jak x > 10 and x < 20
in / not in Operator in
in sprawdza czy element należy do listy, napisu lub innej kolekcji.
wyrażenie warunkowe Jednolinijkowy if (ternary)
Skrócony zapis if-else w jednej linii — często używany w przypisaniach.
💡 Schemat: wartość_true if warunek else wartość_false
list Tworzenie i indeksowanie
Lista to uporządkowany, mutowalny zbiór elementów. Indeksy zaczynają się od 0, ujemne liczą od końca.
metody Metody listy
Najważniejsze metody: append, pop, sort, remove, count, index.
💡 sort() modyfikuje listę w miejscu. sorted(lst) zwraca NOWĄ listę.
zip zip() — łączenie list
zip() łączy dwie listy parami — przydatne gdy masz np. imiona i oceny osobno.
min/max/sum Funkcje agregujące
min(), max(), sum(), len() — klasyka matury informatycznej.
dict Podstawy słownika
Słownik przechowuje pary klucz: wartość. Klucze muszą być unikalne i niezmienne.
💡 Użyj .get(klucz, default) zamiast [klucz] gdy klucz może nie istnieć!
iteracja Iteracja po słowniku
.keys(), .values(), .items() — trzy sposoby przechodzenia po słowniku.
zliczanie Wzorzec zliczania — matura!
Najczęstszy wzorzec na maturze: zlicz wystąpienia elementów za pomocą słownika.
💡 max(d, key=d.get) — klucz z największą wartością w słowniku
dict comprehension Dict Comprehension
Analogicznie do list comprehension — tworzy słownik jedną linią kodu.
def Definiowanie funkcji
Funkcja to nazwany blok kodu, który można wielokrotnie wywołać. return zwraca wartość.
rekurencja Rekurencja
Funkcja wywołuje samą siebie. Kluczowe: przypadek bazowy który zatrzymuje rekurencję.
💡 Bez przypadku bazowego → RecursionError!
lambda Funkcje lambda
Anonimowe funkcje jednolinijkowe. Używane z sorted(), map(), filter().
*args / **kwargs Zmienna liczba argumentów
*args — tuple argumentów pozycyjnych. **kwargs — słownik argumentów nazwanych.

strMetody napisów

Klasy i Programowanie Obiektowe
Klasa to szablon obiektu. Obiekt = dane (atrybuty) + zachowanie (metody). Słowo kluczowe: class
class Nazwa:Definicja klasy
__init__(self,...)Konstruktor — inicjalizacja
selfReferencja do bieżącego obiektu
__str__(self)Reprezentacja tekstowa (print)
__repr__(self)Reprezentacja deweloperska
class B(A):Dziedziczenie po klasie A
super().__init__()Wywołaj konstruktor rodzica
isinstance(obj, Cls)Sprawdź typ obiektu
classPierwsza klasa — Punkt
Konstruktor __init__ jest wywołany automatycznie przy tworzeniu obiektu (p = Punkt(3, 4)).
class Punkt:
def __init__(self, x, y):
self.x = x # atrybut obiektu
self.y = y
↑self — zawsze pierwszy parametr metody, wskazuje na obiekt
↑self.x — atrybut należący do tego konkretnego obiektu
💡 __str__ jest wywoływany przez print() i str() — zawsze go definiuj!
dziedziczenieDziedziczenie — class B(A)
Klasa potomna dziedziczy wszystkie metody i atrybuty klasy rodzica. Może je nadpisać (override).
💡 super() daje dostęp do metod klasy rodzica — używaj w __init__ gdy rozszerzasz konstruktor
metody klasoweKlasa z logiką — Student
Praktyczny przykład klasy z wieloma metodami — przechowuje dane i udostępnia operacje na nich.

Obsługa wyjątków — try / except
Wyjątki to błędy w czasie działania programu. try/except pozwala je przechwycić i obsłużyć zamiast crashować.
try: ... except:Podstawowa obsługa błędów
except ValueError:Zły typ/format danych
except ZeroDivisionError:Dzielenie przez zero
except IndexError:Indeks poza zakresem listy
except KeyError:Brak klucza w słowniku
except FileNotFoundError:Plik nie istnieje
except Exception as e:Każdy wyjątek, nazwa w e
else:Wykonaj jeśli brak błędu
finally:Wykonaj zawsze (np. zamknij plik)
raise ValueError("msg")Rzuć wyjątek ręcznie
try/exceptPodstawowa obsługa błędów
Bez try/except błąd przerywa program. Z obsługą możemy zareagować i kontynuować.
try:
# kod który może się nie udać
except ValueError:
# obsługa konkretnego błędu
else:
# jeśli NIE było błędu
finally:
# zawsze — sprzątanie
💡 finally wykonuje się ZAWSZE — świetne do zamykania plików, połączeń itp.
ValueError/TypeErrorWiele typów wyjątków
Możesz przechwycić kilka różnych typów błędów — każdy w osobnym except.
💡 Kolejność except ma znaczenie — bardziej szczegółowe wyjątki wyżej!
raiseRzucanie własnych wyjątków
raise pozwala celowo wywołać wyjątek gdy dane są nieprawidłowe — lepsza walidacja.

Pliki I/O — odczyt i zapis
Python obsługuje pliki przez open(). Zawsze używaj with open(...) as f: — automatycznie zamyka plik.
open(plik, "r")Otwórz do odczytu (read)
open(plik, "w")Zapis — nadpisuje plik!
open(plik, "a")Append — dopisuje na końcu
open(plik, "r", encoding="utf-8")Polskie znaki!
f.read()Cały plik jako string
f.readlines()Lista linii (z \n)
f.readline()Jedna linia
for linia in f:Iteracja po liniach
f.write(tekst)Zapisz string do pliku
linia.strip()Usuń \n na końcu linii!
symulacjaSymulacja odczytu pliku
W przeglądarce nie możemy otworzyć prawdziwych plików, ale poniżej pokazujemy jak wyglądałby kod i co by zwrócił.
💡 Na maturze dane z pliku to najczęściej liczby w kolejnych liniach — schemat wyżej jest kluczowy!
CSVPrzetwarzanie CSV
CSV (Comma Separated Values) — najczęstszy format danych na maturze. Każda linia = jeden rekord, pola oddzielone przecinkiem.
💡 Wzorzec maturalny: for linia in linie[1:]: — pomiń nagłówek CSV!
wzorzecKod do prawdziwych plików
Tak wygląda prawdziwy kod odczytu pliku — użyj tego wzorca na maturze.
with open("dane.txt", "r", encoding="utf-8") as f:
linie = f.readlines()
for linia in linie:
linia = linia.strip() # usuń \n
↑with — automatycznie zamknie plik, nawet przy błędzie
↑encoding="utf-8" — ZAWSZE dla polskich znaków!
↑.strip() — usuwa znaki nowej linii \n z końca każdej linii
