Plan struktury

Każdy przedmiot = 2 pliki. Spójna konwencja nazewnictwa, identyczna architektura sekcji.

01
Pliki na przedmiot
Każdy przedmiot składa się z dokładnie 2 plików HTML:

[przedmiot]_teoria.html — teoria, słownik, moduły, quiz, fiszki, typing, fokus, notatnik

[przedmiot]_cwiczenia.html — interaktywne zadania z automatycznym sprawdzaniem, izolowane środowisko, baza zadań 500–800+
02
Sekcje w pliku TEORIA
Hero — branding, poziomy, CTA
Roadmapa — checkbox lista tematów z localStorage
Słownik — karty z filtrem + wyszukiwarką
Moduły — pełna teoria z przykładami i błędami
Przykłady — interaktywne przykłady uruchamiane w przeglądarce
Quiz — 20–30 pytań z wyjaśnieniami
Fiszki — 25–40 kart z TTS, shuffle, mastered
Typing Sprint — 15–25 zadań wpisywania
Fokus — tryb z timerem i klawiszami
Notatnik — builder ściągi z exportem .txt
03
Sekcje w pliku ĆWICZENIA
Panel nawigacji — filtr kategorii, poziomy, postęp dzienny
Lista zadań — po lewej, paginowana, z flagami "znam/nie znam"
Edytor — środowisko do pisania kodu/SQL/odpowiedzi
Weryfikacja — automatyczne sprawdzanie (sql.js / eval)
Podpowiedź + Odpowiedź — ujawnianie stopniowe
phpMyAdmin-like panel — podgląd tabel / stanu (dla SQL/Access)
Statystyki — dzienne cele, streak, postęp kategorii
Reset środowiska — per-zadanie izolowane tabele
04
Palety kolorów per przedmiot
■ SQL — cyan (#00d4ff) + teal
■ Python — gold (#ffd43b) + orange
■ Excel — green (#10d48e) + emerald
■ Access — orange (#ff8c42) + amber
■ Fizyka — rose (#f43f5e) + red
■ Matematyka — blue (#3b82f6) + indigo

Tło zawsze: #080c14. Fonty zawsze: Cinzel + Outfit + Fira Code.

Konwencje ViralloEdu

Zasady które obowiązują w KAŻDYM pliku platformy — dla spójności.

🎨 Estetyka

  • Tło zawsze #080c14 (dark navy)
  • Fonty: Cinzel (nagłówki) + Outfit (body) + Fira Code (kod)
  • Każdy przedmiot ma własny kolor akcentu
  • Noise overlay + grid bg na każdej stronie
  • Radius kart: 16–24px. Zawsze border + hover effect

📁 Nazewnictwo plików

  • sql_teoria.html — teoria SQL
  • sqledukacja_v25.html — ćwiczenia SQL
  • python_teoria.html — teoria Python ✓
  • python_cwiczenia.html — ćwiczenia Python
  • virallo_hub.html — ta strona (hub)

🧩 Sekcje teorii (kolejność)

  • Hero → Roadmapa → Słownik → Moduły
  • → Przykłady → Quiz → Fiszki
  • → Typing Sprint → Fokus → Notatnik
  • localStorage: checkboxy roadmapy + mastered fiszki
  • Toast notifications przy akcjach

⚙️ Technologia

  • Zero frameworków — vanilla HTML+CSS+JS
  • Jeden plik = cała aplikacja (self-contained)
  • SQL: sql.js (SQLite w przeglądarce)
  • Python: Skulpt.js (eval w przeglądarce)
  • Wzory: MathJax dla fizyki i matmy

📊 Zawartość bazy zadań

  • Minimum 400–800 zadań per przedmiot
  • Każde zadanie: treść, odpowiedź, poziom, kategoria
  • Zadania DML/DDL: izolowane tabele (setup SQL)
  • Tagi poziomów: podstawowy, średni, zaawansowany, matura
  • Postęp dzienny: cel = 25 zadań

🎓 Zgodność z CKE

  • SQL: standard SQLite + uwagi o Access
  • Python: Python 3.x, standardowa biblioteka
  • Excel: polskie nazwy funkcji (JEŻELI, WYSZUKAJ.PIONOWO)
  • Fizyka/Matma: zgodność z podstawą programową
  • Każde zadanie ma oznaczenie "maturalne" jeśli pochodzi z CKE
